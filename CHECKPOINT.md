마지막 갱신: 2026-07-05

## 현재 위치

M1 구현 완료, M2 독립형 음성 통역기 0.3.0 포터블 빌드와 핵심 실기 검증 완료.

- 60/40/28문장 분리 평가셋 핵심정보 보존 점수 모두 1.000
- 영어·한국어·중국어·스페인어 공개 음성 및 일본어 직원 답변 분기 9/9 통과
- Windows WASAPI PC 재생음 실제 TTS 캡처 통과
- 실행 PC별 입력·출력 장치 GUID 자동 검색, 10초 목록 갱신 및 장치 분리 시 기본값 복구
- 음성 장치 실패 시 2초 간격 자동 재연결, Hy-MT2 장애 시 1회 자동 재시작
- 중복 M2M100/Torch/Transformers를 배포본에서 제외해 2.64GiB → 1.79GiB
- 포터블 EXE 모델 준비, loopback/출력 선택, UI 오류 0건, 종료 후 잔류 프로세스 0건 확인
- 음성을 저장하지 않는 `feedback/corrections.jsonl` 교정 기록 추가
- 단위·보안 테스트 48개 및 정적 검사 통과
- localhost 세션 쿠키, Host·WebSocket Origin 검사, llama API key 적용
- 모델·런타임 SHA-256 검증과 zip 경로 이탈 방지 적용
- 개인 설정·교정 기록은 `%LOCALAPPDATA%\RemotePlusTranslator`, 모델은 EXE 옆에서 절대경로로 로드
- Claude 정적 분석 29개 항목 검토 결과는 `FIXES_REVIEW.md`에 기록

- Whisper-small 463.7MB 다운로드 완료
- M2M100 PyTorch 가중치 약 1.94GB 다운로드 완료
- CTranslate2 INT8 변환 완료, 로딩 약 13초
- 텍스트 번역 지연은 방향별 약 0.3~0.6초
- 영어/스페인어→일본어와 일본어→영어/한국어/중국어 스모크 결과는 대체로 자연스러움
- 한국어/중국어 `배송 주소`가 일본어 `送料アドレス`로 오역됐고 일본어→스페인어 어휘 흔들림 확인
- M2M100을 즉시 확정하지 않고 OPUS/Marian 언어쌍 모델과 비교 후 파인튜닝 여부 결정
- 호텔 도메인 STT 적응 추가: 영어·한국어·중국어·스페인어·일본어 각 32개 용어
- Whisper beam size 1 → 3, 대화 언어별 hotword 선택, 반복 오인식 교정 사전 적용
- `콜을 주세요 → 콜라 주세요`, `진저일 → 진저에일` 사용자 사례 반영
- 입력 언어를 자동/수동으로 선택 가능. 수동 외국어 모드에서도 일본어 직원 답변은 별도 감지
- 호텔 합성 음성 32문장 STT 평균 유사도 0.961
- 4개 외국어→일본어 16문장 핵심 용어 점수 0.625였으며 호텔 용어 안전장치 적용 후 0.958
- 브라우저 대신 pywebview 전용 창을 사용하며 휴대용 EXE 이름은 `RemotePlusTranslator.exe`
- OnStreamLLM Lite를 참고한 어두운 대시보드 UI와 일본어·한국어·영어·중국어·스페인어 화면 전환 완료
- 60문장 확장 회귀와 40문장 홀드아웃, 28문장 2차 검증에서 조정 후 핵심정보 보존 1.000
- 영어/한국어 수동 입력 모드와 일본어 답변 분기 실제 합성 음성 4/4 통과
- 양방향 다국어 호텔 보호 용어와 15개 고위험 일본어 답변 문장집 적용
- OPUS 영어→일본어는 품질 미달로 기각. OPUS 일본어→영어는 좋았지만 다른 언어 영어 피벗에서 오전/오후 및 명사 오역이 생겨 전체 적용 기각
- 보호 용어가 개입한 UI 결과는 주황색으로 표시
- 화면에서 12개 감지 입력 장치와 시스템 기본 장치를 선택하고 캡처를 재시작하는 기능 검증
- 경량 배포본에는 약 0.45GB Whisper와 1.13GB Hy-MT2 Q4만 포함하고 중복 번역 모델은 제외

## 확인된 PC 환경

- Windows, Python 3.13.0
- Intel Core i5-12450H, RAM 16GB
- Intel UHD Graphics
- 기본 마이크/스피커가 sounddevice에서 감지된 이력 있음

## 다음 회차 첫 작업

1. `run.bat`으로 데스크톱 앱 창 실행
2. 영어 1문장 → 일본어 표시 → 일본어 답변 → 영어 TTS 실기 검증
3. 같은 절차를 한국어·스페인어·중국어에 반복
4. 헤드셋과 노트북 스피커 환경에서 TTS 재입력 차단 확인
5. 실제 오류 문장을 회귀 데이터로 저장
6. 30분 연속 왕복 대화 후 지연·메모리·누락 기록

PBX/소프트폰 연동은 위 독립형 왕복 흐름이 안정화된 뒤 M4에서 진행한다.

## 아직 검증하지 못한 위험

- 스페인어·중국어 Windows SAPI 음성이 아직 설치되지 않음
- 실제 사람/전화 잡음 환경에서 합성 음성 0.961 수준이 유지되는지
- 번역 문장의 자연스러움은 용어 보존과 별도로 추가 평가 필요
- 기본 마이크에서 RMS 임계값의 적합성
- 일본어 짧은 답변의 언어 감지 확률

## 중요한 설계 결정

- 첫 안정 버전은 CPU를 기준선으로 삼음. Intel GPU 최적화는 결과 비교 후 도입.
- 일본어 입력은 직전 90초 이내 상대 언어로 답변하는 것으로 해석.
- 일본어 고객 응대는 번역 대상이 아니며, 일본어 감지는 직원 답변 방향 판별에만 사용.
- 모델 준비 실패 시 마이크를 시작하지 않아 반쪽짜리 실행 상태를 피함.
- 사용자 설정은 `config.local.toml`에 두어 기본 설정 업데이트와 분리.
- 상업 콜센터이므로 CC-BY-NC인 NLLB/MMS는 사용 금지. M2M100(MIT)과 SAPI 기준.
- 실제 통화 제품은 고객/상담원 오디오를 별도 채널로 받아야 함. 현재 단일 마이크는 실기 프로토타입.
