# 개발 로드맵

각 단계는 앞 단계가 실제 환경에서 통과한 뒤 진행합니다. 체크박스가 다음 회차의 시작점입니다.

## M1 — 제품형 기반과 왕복 흐름

- [x] 모듈형 오디오/STT/번역/TTS 구조
- [x] 외국어 → 일본어 텍스트 상태 머신
- [x] 일본어 답변 → 직전 상대 언어 번역과 Windows TTS
- [x] TTS 마이크 재입력 차단
- [x] bounded queue, 오류 이벤트, 최근 기록 복원
- [x] 로컬 웹 UI와 조작 API
- [x] 브라우저 주소창 없는 데스크톱 프로그램 창
- [x] 자동 감지/입력 언어 직접 선택과 일본어 답변 예외 처리
- [x] 설치·진단·모델 준비 스크립트
- [x] 모델 없이 실행되는 핵심 단위 테스트

완료 기준: 정적 검사와 단위 테스트 통과. 실제 모델 다운로드 테스트는 M2에서 수행.

## M2 — 독립형 음성 통역기 실기 완성

- [x] 고정 의존성 전체 설치
- [x] Whisper-small/M2M100 다운로드 및 INT8 변환
- [x] 영어·한국어·일본어 합성 음성 32문장 회귀 테스트
- [x] 영어·한국어·중국어·스페인어 호텔 번역 16문장 회귀 테스트
- [ ] 스페인어·중국어 Windows 음성 설치 후 STT/TTS 회귀 테스트
- [ ] 실제 마이크 외국어 입력 → 일본어 자막 확인
- [x] PC 스피커/영상 소리를 WASAPI loopback으로 직접 입력
- [x] 마이크·스테레오 믹스·가상 케이블 입력을 화면에서 선택하고 즉시 재시작
- [ ] 원문·일본어 번역을 실시간 팝업 자막으로 표시
- [ ] 일본어 마이크 입력 → 직전 상대 언어 TTS 확인
- [x] 번역된 TTS를 선택한 스피커/가상 오디오 출력으로 전송
- [ ] TTS가 다시 마이크로 들어가 반복되는지 실기 확인
- [ ] 헤드셋/노트북 스피커 환경 각각 왕복 테스트
- [ ] 실제 마이크 왕복 대화 30분 soak test
- [ ] 발화 종료·잡음 임계값을 이 PC 마이크에 맞게 보정
- [ ] STT/번역/TTS p50·p95 지연 기록
- [ ] 메모리 사용량과 장시간 큐 안정성 확인
- [ ] 영어·한국어·스페인어·중국어 Windows 음성팩 검증

완료 기준: 네 언어가 각각 일본어로 표시되고 일본어 답변이 원래 언어 TTS로 출력됨.

## M3 — 체감상 동시통역

- [ ] 말하는 중간 결과를 latest-only 슬롯으로 인식
- [ ] 확정된 prefix만 번역해 자막 흔들림 억제
- [ ] 최종 발화가 중간 결과를 항상 선점하도록 우선순위 적용
- [ ] 번역 중 문장 순서 보장 및 취소 토큰
- [ ] 입력/출력 채널별 독립 시작·중지와 언어 설정
- [ ] OBS 브라우저 오버레이용 투명 자막 화면

완료 기준: 10초 발화 중 2초 이내에 첫 일본어가 보이고, 최종 결과가 중복되지 않음.

## M4 — 콜센터 시스템 연결

- [ ] 고객 통화 수신 채널과 상담원 마이크 채널 분리
- [x] WASAPI loopback 장치 선택 도우미
- [ ] 헤드셋 없이 사용할 때 AEC/잡음 억제 경로 검토
- [ ] 입력 채널별 방향을 명시해 일본어/외국어 오감지 제거
- [ ] PBX/소프트폰별 오디오 입출력 연결 방식 문서화
- [ ] 상담원별 세션 격리와 장애 시 원래 통화 우회
- [ ] 녹음·전사 보관 정책과 개인정보 마스킹

완료 기준: Zoom/Meet/게임 음성에서 상대 채널만 안정적으로 번역.
