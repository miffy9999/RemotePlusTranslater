# 호텔 내부 상업 운영 안전 기준

이 문서는 RemotePlus Translator를 웹 판매나 불특정 배포가 아니라 호텔 민원실·콜센터 내부에서
사용할 때의 최소 운영 기준이다. 법률의견서가 아니라 실제 사고 가능성을 줄이기 위한 배포·운영
체크리스트다.

## 운영 시작 전 필수

- 호텔의 개인정보 처리 책임자와 시스템 책임자를 각각 지정한다.
- `legal/distributor-info.example.json`을 복사하여
  `legal/distributor-info.local.json`으로 만들고 실제 호텔/운영 법인의 정보만 기입한다.
- 프로그램 제공 법인과 통화 운영 호텔이 다르면 `publisher_*`와 `operator_*`를 구분한다. 같은
  법인이 두 역할을 모두 할 때만 동일 정보를 반복한다.
- 통화 시작 시 AI 음성 인식·번역·합성음성을 사용한다는 안내를 한다.
- 별도 전화 녹음 장치가 있다면 녹음 목적, 보관 기간, 접근 권한, 삭제 절차도 함께 고지·관리한다.
- 직원 교육에서 긴급·의료·알레르기·안전·결제·예약·신원 확인은 번역 결과만 보고 확정하지 않고
  숫자, 날짜, 성명, 부정 표현을 반드시 복창하도록 한다.
- 관리 대상 Windows PC에만 설치하고 일반 사용자가 EXE, 모델, 설정을 교체하지 못하게 한다.

## 매 통화 시

1. 고객 언어와 입력·출력 장치를 확인한다.
2. AI 사용 안내 후 통화를 진행한다.
3. 화면 원문과 일본어 번역을 함께 확인한다.
4. 중요한 내용은 짧게 나누어 다시 질문하고 복창한다.
5. 통화가 끝나면 `履歴を消去`를 눌러 메모리 기록을 지운다.
6. 오인식 교정을 저장할 때 고객 이름, 전화번호, 예약번호, 건강정보를 제거한다.

## 즉시 자동 응답을 중단할 조건

- 언어 선택이 실제 고객 언어와 다르다.
- 반복 단어, 무음, 심한 지연, 음성 장치 끊김 또는 프레임 드롭 경고가 나온다.
- 고객과 직원이 겹쳐 말하거나 전화 음질이 너무 낮다.
- 긴급 신고, 생명·신체 위험, 중대한 알레르기, 카드·계좌 정보, 법적 분쟁이 나온다.
- 고객이 AI 음성 사용을 원하지 않거나 오역을 주장한다.

이 경우 TTS를 끄고 사람이 직접 확인하거나 정식 통역 지원으로 전환한다. 프로그램이 동작한다는
사실을 정확성의 증거로 취급하지 않는다.

## 내부 배포

- 개발 빌드와 운영 빌드를 구분한다. 운영본은 `build.ps1 -CommercialRelease`로만 만든다.
- 운영 PC는 호텔 IT가 신뢰하도록 배포한 내부 코드 서명 인증서를 사용한다. 인증서의 개인키는
  빌드 담당자 외에 복사하지 않는다.
- EXE 하나가 아니라 설치 프로그램 또는 검증된 배포 폴더 전체를 사용한다.
- 배포 전 `signature-report.json`, SBOM, 제3자 라이선스, 렌더링된 EULA와 개인정보 문서를 보관한다.
- 업데이트는 먼저 시험 PC에서 실제 전화 장치로 확인한 뒤 운영 PC에 순차 배포한다.

## 사고 대응

오역이나 개인정보 사고가 의심되면 시각, PC명, 앱 버전, 선택 언어, 영향 통화를 기록하되 통화
내용 자체를 불필요하게 복사하지 않는다. 해당 PC의 자동 응답을 중지하고 책임자에게 보고한 뒤,
관련 로그의 접근을 제한하고 재현·수정·직원 안내가 끝나기 전까지 동일 기능의 재사용을 보류한다.
