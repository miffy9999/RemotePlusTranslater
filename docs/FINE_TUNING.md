# Hy-MT2 호텔 번역 적합화와 후보 모델 승격 기준

## 결론

현재 운영 모델은 상업 사용 가능한 Apache-2.0 라이선스의 Hy-MT2 1.8B Q4_K_M입니다. 7B급
모델은 이 CPU 환경에서 지연이 늘고, NLLB 계열은 비상업 라이선스이므로 운영 후보에서 제외합니다.
새 모델은 단순히 평균 품질이 좋아 보인다는 이유로 교체하지 않고 영어·한국어 양방향 고정
평가셋과 실시간 지연 게이트를 모두 통과할 때만 승격합니다.

`finetune/hotel_en_ko_seed.json`에는 개인정보가 없는 영어·한국어 중심 호텔 회화와 문맥 예시가
학습 40건, 미사용 holdout 8건으로 분리되어 있습니다. 다음 명령은 데이터 누출·중복·카드번호·
전화번호·이메일 패턴을 검사하고 Hy-MT 공식 `messages` JSONL 형식으로 렌더링합니다.

```powershell
.\.venv\Scripts\python.exe scripts\prepare_finetune_data.py
```

생성물은 `cache/finetune/hotel_train.jsonl`과 `hotel_holdout.jsonl`입니다. holdout은 학습이나
프롬프트 조정에 사용하지 않습니다. 초기 40건은 파이프라인과 회귀 안전장치를 준비하는
seed이며, 실제 모델을 상업 배포할 만큼 충분한 학습량이라는 뜻은 아닙니다.

각 번역 카드의 `번역 수정` 버튼으로 번역 결과를 바로 고칠 수 있습니다. 승인한 번역은
`원문 + 원문 언어 + 번역 언어` 기준으로 로컬 번역 메모리에 저장되며, 다음에 같은 문장이
들어오면 AI를 다시 호출하기 전에 즉시 사용합니다. 음성은 저장하지 않고 30일 교정 감사 기록과
최대 2,000개의 승인 번역만 `%LOCALAPPDATA%\RemotePlusTranslator\feedback`에 보관합니다.
이름·전화번호·예약번호·카드번호 같은 개인정보는 교정 메모리에 등록하지 않습니다. 실제 현장
오류가 언어별로 충분히 쌓인 뒤 개인정보를 제거하고 고정 평가셋과 학습셋을 분리합니다.

먼저 영어·한국어 각 방향별로 현장 검수 완료 문장을 최소 100개씩 확보하고 오류를 분류합니다.
파인튜닝 여부는 학습에 쓰지 않은 고정 평가셋의 사람 평가, 숫자·부정·정책 보존, 지연으로
결정합니다. 고객 원음, 이름, 예약번호, 전화번호, 결제정보는 학습 데이터에 넣지 않습니다.

| 오류 | 먼저 할 일 | 파인튜닝 조건 |
|---|---|---|
| 이름·제품명 오인식 | Whisper initial prompt/용어 사전 | 같은 도메인 오디오가 반복적으로 틀리고 전사 데이터가 충분할 때 |
| 짧은 말의 언어 오감지 | 언어 고정·대화 언어 기억 | 별도 언어 식별 데이터가 충분할 때 |
| 전문용어 번역 오류 | 용어집 강제 치환·번역 메모리 | 고품질 병렬 문장이 수만 쌍 이상 있을 때 |
| 문체가 어색함 | 후처리 규칙·더 큰/언어전용 모델 비교 | 자동 지표와 사람 평가에서 지속적으로 열세일 때 |
| 읽기 표기 품질 | 호텔 문구 사전·언어 규칙 보강 | 반복 오독되는 실제 번역문이 충분히 모였을 때 |

## 번역 LoRA 실행 순서

1. Hy-MT2 공식 저장소의 1.8B BF16 원본과 공식 fine-tune 환경을 별도 GPU/Colab에 준비합니다.
2. `hotel_train.jsonl`만 사용해 q/k/v/o projection LoRA를 작은 학습률로 학습합니다.
3. LoRA를 BF16 원본에 병합한 뒤 llama.cpp로 현재와 동일한 Q4_K_M GGUF를 만듭니다.
4. 기본 모델과 후보 모델을 같은 PC, 같은 thread/context 설정, 같은 corpus로 각각 3회 측정합니다.
5. 학습 미사용 holdout에서 핵심 사실·부정·숫자·말투가 한 건도 후퇴하지 않고 모델 경로의
   중앙 지연과 p95 지연도 느려지지 않을 때만 `config.toml`의 모델 경로를 바꿉니다.

무료 Colab은 세션과 GPU가 보장되지 않고 이 PC에는 학습 가능한 NVIDIA GPU가 없으므로, 현재
저장소는 재현 가능한 데이터와 검증 게이트까지만 준비합니다. 검수 데이터가 부족한 상태에서
학습을 강행하는 것은 “상황에 맞춤”이 아니라 합성 오류를 모델에 고정할 위험이 있습니다.

## STT 파인튜닝을 검토할 시점

Whisper 파인튜닝은 특정 억양·현장 잡음·고유명사 때문에 실제 WER이 기준을 넘을 때만 합니다. 학습 데이터는 반드시 고객 사용 환경과 같은 마이크/소음으로 수집하고, 화자와 문장이 겹치지 않는 검증 세트를 분리합니다. 학습 후 faster-whisper용 CTranslate2 형식으로 변환하고 기존 모델과 A/B 테스트해야 합니다.

호텔 STT는 먼저 언어별 hotword와 교정 사전을 적용했습니다. 이후에도 오류가 남으면 실제 호텔 문의 문장을 여러 화자·속도·억양·전화 음질로 녹음하고 정확한 전사를 붙입니다. 개인정보가 포함된 실제 통화 원본을 그대로 Colab에 올리지 않으며, 익명화된 동의 데이터 또는 별도 재녹음 데이터만 사용합니다. 최소 수백 문장으로 평가셋을 먼저 만들고, 학습 효과를 확인할 수 있을 만큼 수천 문장 이상 확보한 뒤 파인튜닝합니다.

번역 파인튜닝은 원문-정답 일본어 쌍과 일본어-상대 언어 쌍을 별도로 평가해야 합니다. 자동 번역물을 다시 학습 정답으로 쓰면 오류가 증폭됩니다. Colab 무료 런타임은 세션과 GPU가 보장되지 않으므로 재개 가능한 체크포인트를 Google Drive에 저장해야 합니다.

모델 라이선스는 파인튜닝해도 바뀌지 않습니다. 현재 기본 Hy-MT2 모델은 Apache-2.0이며,
모델 카드와 저작권 고지, llama.cpp 및 포함 의존성의 고지 파일을 배포물에 유지해야 합니다.
자동 라이선스 검사는 법률 자문을 대체하지 않습니다.
