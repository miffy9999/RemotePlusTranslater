# VPS·현장 피드백 업그레이드 준비안

작성일: 2026-07-19

## 결론

- Sakura VPS는 번역 추론 서버가 아니라 다운로드, 업데이트, 모델 배포를 담당한다.
- 고객 음성, WAV 원본, STT 및 번역은 Windows PC에 남긴다.
- 2GB VPS에 별도 채팅 프로그램이 있어도 RemotePlus를 정적 다운로드 서비스로만 시작하면
  공존할 가능성이 높다. 다만 채팅 프로그램의 실제 메모리와 디스크 사용량을 측정하기 전에는
  API, 데이터베이스 또는 라이선스 서버를 추가하지 않는다.
- WAV 가져오기는 라이브 통화와 분리된 오프라인 작업으로 구현한다. 고객과 직원 음성이 함께
  녹음된 한 개의 WAV를 시간순으로 처리하며, 화자를 확정할 수 없어도 들리는 문장을 번역한다.
- 소음 문제는 RMS 임계값만 일괄 상승시키지 않고 현장 보정, 연속 프레임 확인, 음성 VAD,
  Whisper 무음 신뢰도 검사를 단계적으로 적용한다.

## VPS를 받는 날 확인할 정보

다음 값이 없으면 서버를 변경하지 않는다.

1. 상품명이 `さくらのVPS`인지 확인한다. 공유 `さくらのレンタルサーバ`와 구분한다.
2. Linux 배포판과 버전, 공인 IPv4, SSH 포트, 일반 관리자 계정과 sudo 권한을 확인한다.
3. 디스크 전체/여유 용량과 백업 또는 스냅샷 제공 여부를 확인한다.
4. 함께 실행할 채팅 프로그램의 정확한 제품명, 설치 방식, 데이터베이스, 첨부 파일 크기를 확인한다.
5. 도메인 또는 사용할 서브도메인과 DNS 수정 권한을 확인한다.
6. 기존 방화벽, 리버스 프록시, 사용 중인 80/443 포트와 인증서 발급 방식을 확인한다.
7. 채팅 서비스의 최근 24시간 메모리, CPU, 디스크, 동시 접속자 수를 확인한다.

SSH 개인키나 서버 비밀번호는 저장소, 문서, 채팅 또는 로그에 기록하지 않는다.

## 2GB VPS 공존 합격 기준

RemotePlus 1차 서버 기능은 Caddy가 제공하는 정적 사이트, 설치 파일, 모델 팩,
서명된 릴리스 매니페스트로 제한한다. 이 구성에는 별도 Python API나 데이터베이스가 필요 없다.

채팅 프로그램 기동 후 다음을 실제 측정한다.

- 평상시 `available` 메모리 700MB 이상
- 채팅 사용량이 높은 시간에도 swap thrashing과 OOM kill 없음
- 디스크 여유 공간이 최신 전체 패키지 2개, 모델 팩 2세트, 백업을 보관하고도 20% 이상
- 80/443 포트를 기존 프록시에서 서브도메인으로 분기할 수 있음
- 설치 파일 다운로드 중 채팅 지연이나 오류율 증가가 없음

기준을 충족하지 않으면 VPS 이전을 보류하고 USB 배포를 유지한다. 대안은 다음 중 하나다.

- 기존 채팅 VPS에는 링크와 정적 매니페스트만 두고 대용량 파일을 외부 객체 저장소에 둔다.
- 채팅 프로그램과 RemotePlus 배포 서버를 분리한다.
- VPS 메모리를 상향한다.

## 권장 서버 배치

초기 서브도메인 예시는 다음과 같다. 모두 호텔 고정 공인 IP 또는 호텔-VPS VPN 대역만
접근하도록 제한한다.

- `remoteplus.example.com`: 제품 소개와 설치 안내
- `download.remoteplus.example.com`: 설치 프로그램과 모델 팩
- `update.remoteplus.example.com/stable/manifest.json`: 안정판 릴리스 정보
- `update.remoteplus.example.com/beta/manifest.json`: 시험판 릴리스 정보

릴리스 파일과 모델 팩은 변경하지 않는 버전 경로에 보관한다. 같은 URL의 파일을 교체하지 않는다.
매니페스트에는 최소한 schema, channel, version, published_at, URL, 파일 크기, SHA-256,
필수/선택 업데이트 여부를 기록한다. 매니페스트와 설치 파일은 HTTPS로 받고 설치 파일은
SHA-256, Authenticode 상태, 호텔이 고정한 배포자 인증서 지문을 모두 통과해야 한다.

설치 프로그램과 실행 파일은 Authenticode로 서명한다. HTTPS와 SHA-256만으로는 공격자가
매니페스트와 파일을 함께 바꾸는 상황을 막을 수 없다. 업데이트는 임시 경로에 내려받고 서명과
해시를 검증한 후 교체하며, 실패하면 직전 버전으로 복구한다.

## WAV 대화 가져오기 사양

라이브 파이프라인은 사용자가 선택한 `고객 외국어 → 일본어` 또는 `직원 일본어 → 고객 언어`
방향을 발화 시작 시 고정한다. WAV는 이 라이브 방향과 분리된 오프라인 가져오기 controller와
worker를 사용한다.

### 1차 지원 범위

현장에 이미 존재하는 고객/상담원 혼합 WAV 한 파일을 우선 지원한다. 화자 생체정보를 이용하는
speaker identification이나 별도 화자 분리 모델은 1차 버전에 넣지 않는다.

- 사용자는 고객 언어를 선택한다. 상담원 언어의 기본값은 일본어다.
- 음성 구간이 일본어로 명확하면 `상담원 → 고객 언어`로 번역한다.
- 음성 구간이 선택한 고객 언어로 명확하면 `고객 → 일본어`로 번역한다.
- 언어 또는 화자 판단이 애매해도 문장을 버리지 않고 `화자 미확인`으로 표시한다.
- 미확인 문장은 감지된 언어와 원문을 표시하고, 일본어가 아니면 일본어 번역을 우선 제공한다.
- 사용자는 잘못 분류된 말풍선의 고객/상담원 역할을 나중에 바꿀 수 있다.

스테레오 채널이 분리되어 있거나 고객/직원 WAV 두 파일을 받을 수 있는 경우에는 해당 정보를
자동 분류보다 우선 사용한다. 겹쳐 말한 구간은 한쪽 문장이 누락될 수 있다는 경고를 표시한다.

### 처리 흐름

1. WAV를 로컬에서 열고 PCM 형식, 길이, 채널, 샘플레이트, 파일 크기를 검증한다.
2. 16kHz mono 작업 버퍼로 변환하되 원본을 수정하거나 서버로 전송하지 않는다.
3. VAD로 음성 구간과 타임스탬프를 생성한다.
4. 각 구간을 자동 언어 판별하되 일본어와 선택한 고객 언어만 우선 후보로 사용한다.
5. 판별 신뢰도가 충분하면 해당 언어로 STT하고 반대 언어로 번역한다.
6. 애매한 구간만 일본어/고객 언어 강제 STT 후보의 신뢰도를 비교한다.
7. 시간순으로 고객/상담원/화자 미확인 말풍선을 표시하고 원음 재생과 역할 수정 기능을 제공한다.
8. 필요할 때만 JSON 또는 HTML로 내보내며, 개인정보 포함 경고를 표시한다.

### 안전 한도

- 최초 버전은 WAV만 허용하고 압축 포맷은 후속으로 미룬다.
- 파일 크기와 재생 시간을 제한하고 전체 파일을 여러 복사본으로 메모리에 올리지 않는다.
- 취소, 진행률, 실패 구간 재시도, 앱 종료 시 worker 정리를 제공한다.
- 가져온 WAV와 변환 버퍼는 자동으로 피드백 데이터나 VPS에 업로드하지 않는다.
- 임시 파일은 작업 종료 또는 앱 재시작 시 삭제한다.

## 주변 소음 오인식 개선

현재 RMS VAD는 한 프레임이 시작 임계값을 넘으면 음성을 시작한다. 주변 소음 기반 동적
임계값도 사용하지만 충격음, 문 닫힘, 음악, TV 음성에는 취약하다. Whisper 호출에서도
`vad_filter`를 끄고 세그먼트의 `no_speech_probability`를 폐기 기준으로 사용하지 않는다.

다음 순서로 개선한다.

1. 실행 직후 또는 장치 변경 후 3~5초 주변 소음 보정을 제공한다.
2. 시작 임계값을 넘는 프레임이 2~3개 연속될 때만 발화를 시작한다.
3. 기본 `min_speech_ms`를 현장 녹음으로 평가해 180ms에서 300~450ms 범위로 조정한다.
4. RMS와 별개로 경량 음성 VAD를 통과한 구간만 Whisper에 보낸다.
5. Whisper의 평균 로그 확률과 무음 확률을 기록하고 명백한 무음/환각 결과를 게시하지 않는다.
6. 폐기된 구간 수와 이유를 개인정보 없는 진단 로그에 기록한다.
7. 실제 현장 소음 WAV를 익명화해 회귀 corpus로 유지한다.

Silero VAD를 검토할 경우 공식 MIT 라이선스 저장소의 정확한 릴리스와 ONNX 파일 해시를 고정하고
제3자 고지, SBOM, 모델 영수증에 추가한다. 새 VAD를 넣기 전에는 RMS 변경만으로 음성 첫소리와
작은 목소리가 잘리지 않는지 A/B 측정한다.

## 속도 개선 기준

VPS는 로컬 STT나 번역 추론 속도를 높이지 않는다. 개선은 Windows PC에서 수행한다.

- 거짓 음성 구간을 차단해 불필요한 Whisper 호출을 줄인다.
- 실제 통화 30분에서 STT/번역 p50, p95, 최대값과 폐기율을 측정한다.
- STT와 Hy-MT2의 CPU thread 합이 논리 코어를 과도하게 경쟁하지 않도록 PC별 프로필을 만든다.
- 모델 축소나 beam 하향은 정확도 corpus를 통과할 때만 적용한다.
- WAV 오프라인 모드는 실시간보다 처리량을 우선할 수 있지만 같은 모델 인스턴스에 무제한 병렬
  호출을 보내지 않는다.

현재 남아 있는 타이밍 표본은 STT 1건뿐이므로 최적화 결론을 내리기에 부족하다. 현장 PC에서
소음, 짧은 발화, 긴 발화, 고객/직원 겹침을 포함한 표본을 추가한다.

## 상업용 라이선스와 비용

현재 핵심 모델과 런타임은 MIT, BSD, ISC, Apache-2.0 계열을 중심으로 구성되어 상업 이용 시
통상적인 실행당 로열티는 없다. 다만 라이선스 고지와 라이선스 파일, SBOM, 모델 출처 및 해시를
배포물에 포함해야 한다. 새 모델이나 VAD는 도입 전 별도로 확인한다.

서버비만 발생한다고 확정할 수 없다. 현재처럼 호텔 관리 PC 내부 배포로 제한하면 공개 배포용
코드 서명 인증서 비용은 피할 수 있지만, 배포 방식에 따라 다음 비용이 생길 수 있다.

- 도메인 등록 및 갱신 비용
- 공개 EXE 배포용 신뢰된 코드 서명 인증서 또는 서명 서비스 비용
- VPS 스냅샷, 외부 백업, 객체 저장소 또는 CDN 비용
- 외부 판매 시 개인정보·약관·오픈소스 고지에 대한 전문 법률 검토 비용

호텔 내부 관리 PC에만 배포하고 호텔 IT가 인증서를 신뢰하도록 배포할 수 있다면 자체 서명
인증서를 사용할 수 있다. 불특정 사용자가 사이트에서 내려받는 공개 배포에는 자체 서명을
사용하지 않는다.

운영 설치본은 Microsoft 서명이 유효한 WebView2 Evergreen x64 오프라인 설치 파일을 함께
묶으므로 대상 PC가 인터넷에서 런타임을 따로 내려받을 필요가 없다. WebView2는 Microsoft의
Evergreen 업데이트 정책에 따라 보안 업데이트를 받도록 호텔 PC 정책에서 차단하지 않는다.

WAV 가져오기가 추가되면 제품은 저장된 고객 음성을 취급하게 된다. 녹음 목적, 이용 목적,
접근 권한, 보관 기간, 삭제 방법을 개인정보 문서와 운영 절차에 추가하고, 기본값은 로컬 처리와
명시적 내보내기로 유지한다.

## 구현 순서

1. 완료: 연속 프레임 시작 확인과 Whisper 무음 확률 폐기를 구현했다.
2. 완료: 혼합 WAV 가져오기, 시간순 구간 UI, 자동 언어 방향 판별, 화자 미확인 표시,
   구간 원음 재생, 취소와 임시파일 정리를 구현했다.
3. 완료: 외부 브라우저 대신 WebView2 네이티브 창을 사용하고 백엔드를 숨김 실행한다.
4. 완료: HTTPS·SHA-256·Authenticode·인증서 지문을 검증하는 런처 업데이트 코드와 서명 릴리스
   매니페스트 생성기, 호텔/VPN IP 제한 Caddy 및 다운로드 사이트 템플릿을 준비했다. VPS와
   인증서 전에는 꺼 둔다.
5. 완료: 실제 도메인·CIDR을 검증하는 배포 설정 생성기, 서명 릴리스 게시, SSH 업로드 전
   엄격한 용량 검사, 불변 버전 경로 활성화, 직전 채널 롤백, 운영 HTTPS 종단 검증을 준비했다.
6. 대기: 현장 소음 회귀 WAV와 설정 백업을 확보해 임계값을 A/B 검증한다.
7. 대기: VPS를 받은 날 공존 합격 기준을 측정하고 기존 채팅 프록시에 Caddy 설정을 병합한다.
8. 합격하면 정적 다운로드·업데이트 서버만 먼저 배치하고 실제 도메인과 서명 지문을 설정한다.
9. 역할 수동 수정, 내보내기, 라이선스 API와 데이터베이스는 실제 필요와 서버 여유가 확인된 뒤
   추가한다.
