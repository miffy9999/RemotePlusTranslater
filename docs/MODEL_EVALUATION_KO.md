# 번역 모델 평가 기록

평가일: 2026-07-16

## 운영 기준 모델

- 모델: Hy-MT2 1.8B Q4_K_M
- 라이선스: Apache-2.0
- runtime: 저장소에 SHA-256으로 고정된 llama.cpp b9870 CPU x64
- 설정: CPU 8 threads, context 1024, GPU offload 0
- 호텔 회귀 128건: forward 1.000, reverse 1.000, 실패 0건
- neural model 경로 76건: 중앙 5.502초, p95 6.410초
- 공개 음성 고정 언어 실행 smoke: 영어·한국어·중국어·스페인어 4/4

이 수치는 해당 개발 PC에서 한 번 측정한 기준선이며 다른 호텔 PC의 성능을 보증하지 않습니다.
최종 QA에는 문맥 회귀 8건도 추가해 이후 보고서의 표본 수가 136건으로 늘어납니다.

## 공식 1.25-bit 후보

- 모델: `tencent/Hy-MT2-1.8B-1.25Bit-GGUF`
- 파일: `Hy-MT2-1.8B-1.25Bit.gguf`
- 크기: 461,860,800 bytes
- SHA-256: `CC497FE8F033B52B3B8B00A7669E9661435432F9D4CD43F7ED24400C01507A93`
- 라이선스: Apache-2.0
- 결과: 운영 runtime에서 tensor type 42를 지원하지 않아 로드 실패

후보를 로드하기 위해 운영 runtime을 검증 없이 교체하면 현재 Q4의 속도·안정성 기준이 무효가
됩니다. 따라서 1.25-bit 후보는 운영에 채택하지 않았고, 기존 Q4와 runtime을 유지합니다. 추후
공식 Windows CPU runtime 조합이 명확해지면 별도 폴더에서 Q4 기준선을 다시 측정한 뒤 동일한
품질·속도 승격 게이트를 적용합니다.

## 모델 선택 결론

Hy-MT2 7B는 CPU 지연 증가가 명확해 “속도 저하 없음” 조건에 맞지 않습니다. NLLB 계열은
비상업 용도 제한 때문에 제외합니다. 일반 목적 소형 LLM은 이 고정 호텔 corpus에서 현재 전용
번역 모델보다 낫다는 근거가 없어 교체하지 않습니다. 현재는 Q4 모델을 유지하면서 결정적
상담 문구, 한 번의 호출에 담는 제한 문맥, 사람 검수 LoRA 후보를 함께 사용하는 구성이 가장
안전합니다.
